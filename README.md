# Customer Retail ML Model Comparison

## Project Title
Using Customer Retail Dataset to Implement Various Machine Learning Models and Compare Them

## Project Overview
This project was developed as part of the DevTown Machine Learning bootcamp. The main objective of this project is to build and compare multiple machine learning models using a customer retail dataset.

The project uses customer retail data features such as Quantity, UnitPrice, and Country to train and evaluate different supervised learning models.

## Dataset Features Used
The following dataset features were used:

- Quantity
- UnitPrice
- Country

## Machine Learning Models Used
Three machine learning models were implemented and compared:

1. Logistic Regression
2. Decision Tree Classifier
3. K-Nearest Neighbors (KNN)

## Project Workflow
The project follows a complete machine learning workflow:

1. Load customer retail dataset using Pandas
2. Handle missing values
3. Encode categorical data using LabelEncoder
4. Visualize customer data using Matplotlib
5. Split the dataset into training and testing sets
6. Train Logistic Regression model
7. Train Decision Tree Classifier model
8. Train K-Nearest Neighbors model
9. Evaluate models using accuracy score and confusion matrix
10. Compare model performances using graphs

## Evaluation Metrics
The models were evaluated using:

- Accuracy Score
- Confusion Matrix

## Visualizations
The project includes the following visualizations:

- Customer Distribution Graph
- Model Accuracy Comparison Graph

## How to Run the Project

1. Clone this repository or download the ZIP file.
2. Open `Final_ML_Project.ipynb` using Jupyter Notebook or Google Colab.
3. Make sure the dataset file `customer_retail_dataset.csv` is in the same folder.
4. Run all cells in the notebook.
5. View the model accuracy scores, confusion matrices, and graphs.

## Conclusion
This project demonstrates how different machine learning algorithms perform on the same customer retail dataset. By comparing Logistic Regression, Decision Tree Classifier, and KNN, we can identify which model performs better based on accuracy and confusion matrix results.

This project helped me understand the complete machine learning workflow, including preprocessing, model training, evaluation, and performance comparison.
